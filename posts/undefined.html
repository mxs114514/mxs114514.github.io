网络基础TCP/IP

TCP/IP 是互联网相关的各类协议族的总称

最重要的思想：应用层、传输层、网络层和数据链路层，接口连接（抽象还在追我）

​ 应用层决定了向用户提供应用服务时通信的活动。（FTP，DNS）
传输层对上层应用层，提供处于网络连接中的两台计算机之间的数据传输。（TCP，UDP）
网络层用来处理在网络上流动的数据包。数据包是网络传输的最小数据单位。该层规定了通过怎样的路径（所谓的传输路线）到达对方计算机，并把数据包传送给对方
链路层用来处理连接网络的硬件部分。（物理、硬件）

URI和URL

URI（统一资源标识符Uniform Resource Identifier ）/ URL（统一资源定位符Uniform Resource Locator）

​ 概念：URI是一个用于标识某一资源的字符串，可以唯一地指代互联网上的任何资源。是一种广义的概念，包含了多种资源标识方法。它是一个通用的术语，旨在通过特定的字符串格式来识别资源，不管资源是否可直接访问或已知其具体位置。

​ 组成：RI通常由两部分组成。
方案（Scheme）：指示访问资源的协议或方法，通常以 : 分隔。例如，http://，ftp://，mailto：等。
资源路径（Path）：指定了资源的具体位置或标识符，这部分可以是一个文件路径、数据库记录标识符等。

​ 类型：
URL（统一资源定位符）：是一种特殊类型的URI，它不仅标识资源，还指定了资源的位置以及访问该资源的方法（例如，HTTP、FTP等）。
URN（统一资源名称）：是一种URI，它唯一标识某个资源，但不指定资源的位置。URN的目的是标识资源，而不涉及如何获取该资源。

​ 总结：
URI 是一个广义的术语，用于标识资源，可以是 URL 也可以是 URN。
URL 是 URI 的一个子集，专注于描述资源的定位和访问方法。当你访问网站、下载文件、发送HTTP请求时，实际使用的都是URL。
URN 是 URI 的另一个子集，它提供了资源的永久标识，但不涉及资源的获取方式或位置。
